# 對齊方法索引

## 概述

對齊方法用於讓模型輸出符合人類偏好，通常在 SFT 之後進行。

## 快速選擇

| 場景 | 推薦方法 | 理由 |
|------|----------|------|
| 一般偏好對齊 | ORPO | 無需參考模型，效率高 |
| 精確偏好控制 | DPO | 穩定、可預測 |
| 嘈雜反饋數據 | KTO | 不需要配對數據 |
| 資源受限 | SimPO | 簡化版，輕量 |

## 方法比較

| 方法 | 需要配對 | 需要參考模型 | 訓練效率 | 穩定性 |
|------|----------|--------------|----------|--------|
| **ORPO** | 是 | 否 | ⭐⭐⭐ | ⭐⭐ |
| **DPO** | 是 | 是 | ⭐⭐ | ⭐⭐⭐ |
| **KTO** | 否 | 否 | ⭐⭐⭐ | ⭐⭐ |
| **SimPO** | 是 | 否 | ⭐⭐⭐ | ⭐⭐ |
| **GRPO** | 組內排序 | 否 | ⭐⭐⭐ | ⭐⭐ |

## 資料格式

所有配對方法需要 chosen/rejected 對：

```jsonl
{"prompt": "問題", "chosen": "好答案", "rejected": "壞答案"}
```

## 何時需要對齊？

**需要對齊**:
- 需要精確控制輸出風格
- 有明確的好/壞答案對比
- SFT 後輸出品質不穩定

**不需要對齊**:
- 分類任務（SFT 通常就夠）
- 沒有偏好數據
- 輸出格式簡單且固定
